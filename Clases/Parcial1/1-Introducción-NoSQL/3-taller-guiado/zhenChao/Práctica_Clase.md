### Ejercicio 1: Gestión Universitaria
**Base de datos Relacional (SQL).**
Piénsalo así: una universidad es pura burocracia y orden. No puedes permitirte errores como que un alumno tenga nota en una asignatura que no existe. SQL es estricto y cuadrado; si las reglas no se cumplen, no deja guardar nada. Es perfecto para mantener ese orden militar en matrículas y notas.

### Ejercicio 2: App Móvil
**Base de datos Clave-Valor (NoSQL).**
Aquí necesitamos velocidad pura, como un guardarropa: das tu ticket (ID) y te dan tu abrigo (datos) al instante, sin buscar en mil bolsillos. Además, estas bases tienen una función genial que "autodestruye" los datos viejos, ideal para borrar sesiones de usuario inactivas sin que tengas que hacerlo tú a mano.

### Ejercicio 3: Sensores Industriales
**Base de datos de Series Temporales.**
Imagínate intentar anotar la temperatura de mil máquinas cada segundo en una libreta normal; te volverías loco. Estas bases de datos están diseñadas específicamente para "tragar" millones de datos por segundo sin atragantarse y para responder rápido a preguntas de tiempo, como "¿qué pasó justo hace 10 minutos?".

### Ejercicio 4: Sistema de Contenidos (CMS)
**Base de datos Documental (NoSQL).**
El contenido en internet es caótico: un artículo tiene vídeo, otro solo texto y otro una galería de fotos. SQL odia el desorden, pero una base documental es como una caja flexible: puedes meter lo que quieras dentro (un "documento") sin importar si es diferente al anterior. Es libertad total para el diseño.

### Ejercicio 5: Banco
**Base de datos Relacional (SQL).**
Con el dinero no se juega. Aquí la prioridad es la seguridad, no la velocidad. SQL tiene algo llamado transacciones ACID, que es básicamente un seguro: si se corta la luz a mitad de una transferencia, el dinero no desaparece; la operación se cancela por completo y vuelve a tu cuenta. O se hace todo bien, o no se hace nada.

### Ejercicio 6: Red Social
**Base de datos de Grafos.**
Las redes sociales son como una telaraña gigante de contactos. Si usas una tabla normal para buscar "amigos de amigos de mis amigos", el sistema explota. Las bases de grafos están hechas para esto: conectan puntos (personas) con líneas (relaciones) y saltan de uno a otro rapidísimo, igual que hacemos nosotros al cotillear.

### Ejercicio 7: Buscador Interno
**Base de datos Vectorial.**
Los ordenadores suelen ser muy literales, pero aquí queremos que entiendan el *significado*. Estas bases transforman las palabras en números (vectores) que representan conceptos. Así, si buscas "coche", el sistema entiende que "automóvil" es casi lo mismo y te lo muestra, aunque no hayas escrito esa palabra exacta.

### Ejercicio 8: Sistema Corporativo
**Base de datos Relacional (SQL).**
Las empresas tradicionales son predecibles: empleados, departamentos y jefes. Esa estructura casi nunca cambia. No hace falta reinventar la rueda; SQL es el estándar de toda la vida, funciona perfecto para datos ordenados y es lo que mejor se lleva con los programas de gestión empresarial típicos.
